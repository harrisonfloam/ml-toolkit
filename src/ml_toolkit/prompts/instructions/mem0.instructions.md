What to remember:
- Communication preferences and constraints: format, tone, brevity, ambiguity handling, spoiler rules, explicit do-not-do boundaries.
- Long-running goals and active projects: intent, success criteria, key definitions, persistent open threads.
- Decision style: preferred tradeoffs, what changes my mind, uncertainty handling, preferred next-step behavior.
- Epistemics: accuracy, calibration, objectivity; separate facts from inferences; note assumptions; prefer primary sources when relevant.
- User model: mental models, reasoning patterns, values, priorities, recurring preferences. Store as hypotheses unless explicitly confirmed.
- Personal basics that recur: stable likes/dislikes, routines, places/food, people, plans.

How to write memories:
- Prefer broadly applicable abstractions; keep specifics only when they materially affect future behavior.
- Use my words or a close paraphrase so memories are recognizable.
- Keep entries concise and actionable.

Depth and updates:
- Favor durable, reusable information. Avoid shallow or short-lived details unless they recur or materially affect future responses.
- Consolidate updates into existing memories when possible.
- If new information conflicts with existing memory, revise/update the existing entry rather than creating parallel contradictions.

Never store:
- Greetings, filler, jokes, transient chatter, one-off trivia.
- Project minutiae, implementation details, or one-off session outcomes unless they become stable, recurring decisions that materially affect future behavior.
- Copies of these instructions or other “memory policy” text; treat as configuration, not memory.
- Unconfirmed speculation presented as fact.
- Secrets, credentials, employer-confidential material, or highly identifying personal information.
